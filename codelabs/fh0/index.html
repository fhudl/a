What is a Developer Platform?

When a product exposes some or all of it's feature to an external developer or company in the form of APIs or SDK to create an app, a developer platform enables the app to run inside the product.

Like most of the companies, freshworks offers a platform for the developers to develop apps for the platform.

Developers can build apps for the following freshworks products

  1. Freshdesk
  2. Freshservice
  3. Freshsales
  4. Freshchat
  5. Freshconnect
  6. Freshcaller

This course covers Freshworks app sdk, and basics of app development,

The content is intended for developers with working knowledge of HTML, CSS, and JavaScript.

Why Apps?

The Freshworks suite of products are as feature rich but apps are a necessity to bridge the gap between the niche requirements of customers and the built-in features of a product.

What do I need to create an app?

To get started with the freshworks app, the developer is expected to have working knowledge of the following,

  1. Freshworks app SDK
  2. HTML
  3. CSS
  4. JavaScript
  5. Basic shell/unix commands
  6. Account for the product, to which the app is intended for

Freshworks Developer Kit (FDK), is a command line interface that helps a developer to create apps for the platform. The developer can perform the following actions with the FDK

  • Create a Freshworks app
  • Run an app
  • Validate the app using ESlint
  • Pack or /bundle

Files Explained

app.js

app.js is where the business logic of the app resides in

template.html

This file contains the HTML required for the app's UI which is rendered in an IFrame.

iparams.json

This file contains all the installation parameters which you will have to specify/set when you install the app, like API keys

iparam_test_data.json

This file contains the installation parameter values which you will specify to test the app on your machine.

manifest.json

This file contains details about your app and its characteristics.

Manifest.json

manifest.json file is one of the important files, the file contains the data that defines the app, i.e

product for which the app is developed

The location where the app has to be loaded,

Which HTML file has to be loaded inside the iframe

Introduction to Freshworks App SDK features

Freshworks App SDK is equipped with features and API's that helps the developer in building apps in an efficient and less time consuming way

FDK features are classified into basic and advanced features,

Basic features

Data APIs

Used to fetch data from the product(Freshdesk, Freshservice)

Events API

Enable developers to react to events that occur in the user interface of a page. This includes button clicks, changes, and updates to field values

Interface API

Used to trigger user interface actions on a page. With these APIs, an app can control the visibility of ticket properties, hide or disable buttons, and show dialog boxes and notifications.

Advanced Features

Data Storage

Data storage feature helps the developer to store data in the platform in JSON format upto 8kb on per account AND per app basis

Request API

Request API is used to make https calls

Instance API

Instance API is used to manipulate an instance of the app if multiple instances of the app are available

OAuth

In addition to basic security, through Iparams, Freshworks platform also supports OAuth for authorization

Though an app can use any of the above mentioned APIs or in combination, sdk APIs are not as same the product core APIs, To access product APIs like ticket API, contact API or agent API from the app, the app needs to make use of API key like an external app
